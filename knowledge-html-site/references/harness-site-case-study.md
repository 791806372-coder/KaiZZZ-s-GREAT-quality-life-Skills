# Harness Engineering Case Study

Use this reference when the target quality bar is the Harness Engineering site style.

## What Makes It Work

- Clear product-like framing: one named concept, one target reader, one memorable definition, one visible knowledge map.
- Multi-page depth: the homepage sells the map; subpages carry the actual teaching load.
- Strong information architecture: every page has top nav, left TOC, section numbers, page title, lead, sections, source links, and previous/next navigation.
- Cinematic but controlled visual language: black background, cream text, serif italic accents, mono labels, large negative space, subtle noise, soft borders, low-contrast surfaces.
- Reusable components: top nav, footer, CTA button, citation link, concept pill, tile, grid, TOC, simulator panel, comparison card, flow step.
- Evidence posture: citations are inline and visible, not hidden in a bibliography only.
- Interactive explainers: step simulators and process panels make abstract systems feel operational.
- Motion discipline: reveal animations and glows add depth without hiding content.

## Deconstructed Skill Stack

1. Source packet research: identify primary sources and what each source supports.
2. Concept compression: define the topic in one sentence and one formula/diagram.
3. Reader journey design: order boards from "what" to "why" to "how" to "resources".
4. Editorial hierarchy: use short bilingual labels, crisp Chinese headings, and explanatory body copy.
5. Evidence mapping: attach citations at the claim level and collect all sources in a resource board.
6. Visual system design: tokens, typography, surface grammar, hover states, and motion rules.
7. Component system: convert repeated teaching patterns into reusable HTML/CSS structures.
8. Explainer simulation: represent a process as steps, active states, progress, and a final output.
9. Responsive adaptation: preserve navigation and scanning on small screens.
10. QA and publishing: check local links, JS syntax, broken sources, mobile layout, and source/copy consistency.

## Fixed Production Flow

1. Source map: collect references, facts, quotations to avoid, and open questions.
2. Board map: choose 5-8 boards and define the learning outcome of each.
3. Homepage storyboard: hero, definition, mechanism, contrast, board cards, footer.
4. Subpage template: TOC, page header, sections, tiles/tables/simulators, source links, next/previous.
5. Design tokens: colors, fonts, spacing, radius, borders, shadows, motion.
6. Component inventory: nav, citation, button, card/tile, concept pill, TOC, flow, simulator.
7. Build shared assets: CSS/JS first, then pages.
8. Content fill: add copy, diagrams, citations, and source library.
9. QA: syntax, local links, browser render when available, responsive, source truthfulness.
10. Package: note any reusable patterns and update the skill/checklist.
